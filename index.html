Continuous Integration

"Continuous Integration is a software development practice where members of a team integrate their work frequently" --Martin Fowler

Advantages

  • Continously integrate new code with the existing code repository
  • All tests run at each integration cycle
  • Immidiate Feedback of code quality through Metrics
  • Constant availability of a deployable source
  • Increased Visibility

Continuous Integration

  • Continuous Integration without tests does not work
  • Start developing tests
    • Write tests for each new identified bug
    • Develop tests for each new feature
